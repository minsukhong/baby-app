# 아기 케어 앱 - 기획서
> AI 작업 규칙: 이 파일을 먼저 읽고, 기획 내용을 완전히 이해한 뒤 구현 방향을 개발자에게 설명하고 승인받은 후 코드 작업 시작.

---

## 서비스 한 줄 정의

> "신생아 부모가 놓치는 정부 혜택을 단 하나도 빠짐없이 챙기게 해주는 맞춤형 퀘스트 앱"

---

## 핵심 로직

```
[사용자 입력 (온보딩)]
  출산 상태 + 날짜
  + 아기 수 + 출생 순서
  + 지역
  + 직업 상태
        ↓
[시스템 계산]
  현재 몇 단계인지 자동 판단
  + 다태아 여부 → 혜택 금액 자동 변경
  + 지역 → 지자체 혜택 필터링
  + 직업 상태 → 해당 혜택만 노출
        ↓
[대시보드]
  맞춤 혜택 리스트 (퀘스트 형식)
  + 완료/미완료 체크
  + 캐릭터 성장 (아기 이모지)
  + 진행률 바
```

---

## 혜택 단계 구조

| 단계 | 시점 | 주요 내용 |
|------|------|----------|
| 1단계 | 임신 확인 ~ 출산 전 | 국민행복카드, 임신기 근로단축, 보건소 혜택 등 |
| 2단계 | 출산 직후 | 첫만남이용권, 산후조리비, 산후도우미 등 목돈 지원 |
| 3단계 | 육아기 | 부모급여, 아동수당, 육아휴직급여 등 매월 지급 |
| 별도 | 워라밸 | 육아기 근로단축 등 직장인/자영업자만 해당 |

## 온보딩 설계 (확정)

### 수집 항목

| # | 항목 | 선택지 | 쓰임새 |
|---|------|--------|--------|
| 1 | 출산 상태 | 임신중 / 출산함 | 현재 단계 판단 |
| 2 | 날짜 | 예정일 or 생일 | D-day 계산, 단계 자동 이동 |
| 3 | 아기 수 | 1명 / 쌍둥이 / 3명↑ | 혜택 금액 자동 분기 |
| 4 | 출생 순서 | 첫째 / 둘째 이상 | 첫만남이용권 등 금액 분기 |
| 5 | 지역 | 당근마켓 단위 | 지자체 혜택 + 산후조리원 필터 |
| 6 | 직업 상태 | 직장인 / 자영업 / 무직 | 육아휴직급여 등 해당 혜택 노출 여부 |

## 캐릭터 시스템 (확정)

### 구조
- **부모 캐릭터** 👨‍👩‍👧 → 처음부터 끝까지 부모 모습 고정
- **아기 캐릭터** 🥚 → 혜택 완료율에 따라 성장

### 아기 성장 단계

| 완료율 | 아기 캐릭터 | 상태 메시지 |
|--------|------------|------------|
| 0% | 🥚 | "아직 알 속이에요" |
| 20% | 🐣 | "껍질을 깨고 있어요" |
| 40% | 🐥 | "세상에 나왔어요!" |
| 60% | 🐤 | "쑥쑥 크고 있어요" |
| 80% | 🐔 | "건강하게 자라는 중" |
| 100% | 👶 | "혜택 덕분에 건강해요!" |

## 혜택 DB 설계 방향 (미확정 항목 포함)

### 확정
- DB에 저장, 관리자 페이지에서 수정 가능
- 연도별 금액 변경 대응 (2026년 기준으로 시작)
- 단태아 / 다태아 / 지역별 분기 처리

### 탐색 필요
- 정부 공식 데이터 출처 (복지로, 정부24, 공공데이터포털 API 여부)
- 지역 단위 구조 (당근마켓 레퍼런스 확인 필요)
- 산후조리원 데이터 출처 미정
