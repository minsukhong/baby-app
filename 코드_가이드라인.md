## 보안 지침 (모든 코드에 적용)

### 절대 GitHub에 올리면 안 되는 것
| 항목 | 저장 위치 | 비고 |
|------|----------|------|
| Google OAuth Client ID/Secret | `.env` | `.gitignore`에 포함 |
| Kakao REST API Key | `.env` | `.gitignore`에 포함 |
| JWT Secret Key | `.env` | `.gitignore`에 포함 |
| DB 비밀번호 | `.env` | `.gitignore`에 포함 |
| 관리자 계정 정보 | DB 시드 (암호화) | 코드에 평문 금지 |

### .env 파일 구조
```
# 절대 GitHub에 커밋하지 말 것
backend/.env         ← 백엔드 환경변수
frontend/.env.local  ← 프론트엔드 환경변수
```

### 코드 보안 원칙
1. **비밀번호** → bcrypt로 해싱, DB에 평문 저장 절대 금지
2. **개인정보** → 전송 시 HTTPS, DB 저장 시 필요한 것만 최소 저장
3. **JWT 토큰** → 만료 시간 설정 필수 (access: 1h, refresh: 7d)
4. **API 키** → 코드에 하드코딩 금지, 반드시 환경변수로 참조
5. **SQL Injection** → ORM(SQLAlchemy) 사용, raw query 금지
6. **CORS** → 허용 도메인 명시적으로 지정
7. **소셜 로그인 키** → 서버사이드에서만 사용, 프론트에 노출 금지

## 배포 아키텍처 (확정)

| 서비스 | 역할 | 이유 |
|--------|------|------|
| **Vercel** | Frontend (Next.js) | Next.js 전용, CDN 최강, 무료 |
| **Railway** | Backend (FastAPI) | 무료 $5 크레딧, 간단한 배포 |
| **Supabase** | DB (PostgreSQL) | 자동 백업, 파일 저장, 소셜 로그인 내장 |

### 확장 전략
```
현재 ~ 1만 사용자  → Vercel + Railway + Supabase (무료)
10만 사용자 이상   → AWS/GCP 마이그레이션 고려 (MSA 구조라 부분 전환 가능)
```

### 배포 환경변수 위치
| 변수 | 위치 |
|------|------|
| NEXT_PUBLIC_API_URL | Vercel 환경변수 |
| DATABASE_URL | Railway 환경변수 |
| JWT_SECRET | Railway 환경변수 |
| SUPABASE_URL / KEY | Railway + Vercel 환경변수 |

## AI 작업 규칙

1. **코드 작업 전 반드시 이 파일 읽기**
2. **구조 변경 시 개발자 승인 후 진행**
3. **작업 완료된 파일은 상태를 ✅로 업데이트**
4. **새 파일 추가 시 이 문서 파일 구조에 반영**
5. **새 개발 단계 시작 시 "현재 진행" 섹션 업데이트**
6. **MSA 구조 (frontend/backend/db 분리) 절대 변경 금지**
7. **환경변수(.env) 파일 내용을 코드에 하드코딩 금지**
8. **민감 정보가 포함된 파일은 .gitignore 확인 후 작업**
